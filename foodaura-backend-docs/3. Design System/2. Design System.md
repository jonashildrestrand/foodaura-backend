# Foodaura Design System

Warm, grounded, kitchen-not-clinic. The system lives in two CSS files and a set of Go templates. Everything is rendered server-side — no JavaScript framework, no build step.

---

## File Structure

```
static/
  colors_and_type.css   ← design tokens (colors, type, spacing, motion, radii)
  app.css               ← component + layout styles (imports tokens)
  logo.svg
  logo-mark.svg
  logo-on-dark.svg

templates/
  _layout.gohtml        ← base HTML shell (loaded last by every page template)
  partials/
    _sidebar.gohtml     ← nav rail with logo, links, notification badge, user chip
    _topbar.gohtml      ← page header (eyebrow / title / actions)
    _recipe_card.gohtml ← recipe card + _tag + _toast defines
    _slot_card.gohtml   ← filled and empty meal slot cards
    _nutrition_bar.gohtml
    _member_chip.gohtml
    _invite_modal.gohtml
  pages/
    plan.gohtml
    discover.gohtml
    household.gohtml
    shopping.gohtml
    notifications.gohtml
    recipe.gohtml
    settings.gohtml
    landing.gohtml
    onboarding.gohtml

internal/vm/types.go    ← all view-model structs
```

## Typography

### Typefaces

| Variable | Stack | Use |
|---|---|---|
| `--font-display` | Fraunces, Georgia, serif | H1–H3, hero text, card titles |
| `--font-body` | Plus Jakarta Sans, system-ui, sans-serif | All body copy, labels, nav |
| `--font-mono` | JetBrains Mono, Menlo | Quantities, ingredient amounts |

Loaded from Google Fonts. To self-host, replace the `@import` in `colors_and_type.css` with `@font-face` blocks.

## Icons

[Lucide](https://lucide.dev) (outline, ~1.6px stroke). Loaded from CDN in `_layout.gohtml`. Icons are activated by `lucide.createIcons()` on DOMContentLoaded.

Usage in templates: `<i data-lucide="icon-name"></i>`

Common icons used:
- `calendar-days` — Meal plan nav
- `utensils` — Discover nav
- `list-checks` — Shopping list nav
- `users` — Household nav
- `bell` — Notifications nav
- `settings-2` — Settings nav
- `check` — Schedule cells, completed steps
- `plus` — Empty slot
- `send` — Invite button
- `x` — Close modal

## Components

### Buttons — `.fa-btn`

```html
<a class="fa-btn fa-btn--primary" href="…">Label</a>
<button class="fa-btn fa-btn--secondary" type="submit">Label</button>
<button class="fa-btn fa-btn--ghost">Label</button>
<button class="fa-btn fa-btn--primary fa-btn--sm">Small</button>
```

Icon buttons:
```html
<button class="fa-iconbtn"><i data-lucide="x"></i></button>
```

### Tags — `.fa-tag`

```html
<span class="fa-tag">Neutral</span>
<span class="fa-tag fa-tag--affirm">Sage</span>
<span class="fa-tag fa-tag--brand">Terracotta</span>
```

### Avatars — `.fa-avatar`

```html
<span class="fa-avatar fa-avatar--p">JH</span>   <!-- primary tint -->
<span class="fa-avatar fa-avatar--s">TN</span>   <!-- sage tint -->
<span class="fa-avatar fa-avatar--lg fa-avatar--p">JH</span>
```

Tint values match `MemberVM.Tint`: `"p"` (terracotta) or `"s"` (sage).

### Cards — `.fa-card`

```html
<div class="fa-card">…</div>
```

### Image Thumbnails — `.fa-thumb`

Aspect-ratio placeholder until real recipe images arrive.

```html
<div class="fa-thumb fa-thumb--peach"></div>
<div class="fa-thumb fa-thumb--sage"></div>
<div class="fa-thumb fa-thumb--oat"></div>
```

### Nutrition Bar — `_nutrition_bar`

Rendered via `{{ template "_nutrition_bar" .Nutrition }}`. Expects `NutritionVM`.

### Toggles, Segmented Controls, Inputs

Used in settings and forms. See `_set_control` define in `settings.gohtml` and CSS in `app.css` (`.fa-toggle`, `.fa-seg`, `.fa-input`).

## Page Shell & Chrome

`_layout.gohtml` is the base HTML shell. Every page template ends with:

```gohtml
{{ template "_layout" . }}
```

The `Chrome.ShowSidebar` field on every VM controls whether the sidebar renders:

```
Chrome.ShowSidebar = true   → .fa-app grid (sidebar + main)
Chrome.ShowSidebar = false  → bare main (landing, onboarding, login)
```

The sidebar requires a fully populated `SidebarVM` on `.Sidebar`.

## View-Model Pattern

All VMs live in `internal/vm/types.go`. Every page VM embeds `BaseVM`:

```go
type BaseVM struct {
    Chrome  ChromeVM
    Sidebar SidebarVM
    Toast   *ToastVM // nil = no toast
}
```

Controllers populate a VM, then call `tmpl.ExecuteTemplate(w, "_layout", vm)`.

The `inc` template function (1-indexed counter) is used in `recipe.gohtml` and `onboarding.gohtml` and must be registered as a custom function:

```go
tmpl.Funcs(template.FuncMap{"inc": func(i int) int { return i + 1 }})
```

---

## Template Loading

Parse all templates at startup so errors surface early:

```go
tmpl := template.Must(
    template.New("").
        Funcs(template.FuncMap{"inc": func(i int) int { return i + 1 }}).
        ParseGlob("templates/**/*.gohtml"),
)
```

Or parse `_layout.gohtml`, all partials, then the specific page template per-request.

## Slots & Tones

`SlotVM.Tone` and `RecipeCardVM.Tone` pick the thumbnail gradient:

| Tone | CSS class | Visual |
|---|---|---|
| `"peach"` | `.fa-thumb--peach` | Warm orange gradient |
| `"sage"` | `.fa-thumb--sage` | Green gradient |
| `"oat"` | `.fa-thumb--oat` | Neutral golden gradient |

Rotate tones based on meal type or recipe category to add visual variety without real images.
