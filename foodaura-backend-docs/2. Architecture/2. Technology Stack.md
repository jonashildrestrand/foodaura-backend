# Technology Stack

## Backend

| | |
|---|---|
| **Language** | Go |
| **Architecture** | MVC with server-side rendering. Controllers handle HTTP requests. Models call MariaDB stored procedures and return data structs. Views are Go templates. Business logic lives in stored procedures, not in the Go layer. |
| **Rendering** | Go `html/template` — the backend renders complete HTML pages and delivers them directly to the browser. There is no separate frontend build step and no client-side framework. |

## Database

| | |
|---|---|
| **Engine** | MariaDB |
| **Role** | Single source of truth for all application data: users, households, nutritional profiles, recipes, meal plans, and shopping lists. Also the home for domain logic. |
| **Domain logic** | Stored procedures implement business rules shared across consumers — recipe scaling, shopping list derivation, meal plan generation, and nutritional calculations. Both the backend and the AI service call stored procedures rather than re-implementing logic in application code. |
| **Migrations** | Flyway, running as a one-shot Docker container. Versioned migrations (`V__`) for table DDL and grants; repeatable migrations (`R__`) for stored procedures and triggers. The backend has no involvement in database provisioning. |

## Networking & Security

| | |
|---|---|
| **Reverse proxy** | Nginx |
| **TLS** | Nginx terminates HTTPS. The backend communicates over plain HTTP internally. |
| **mTLS** | Each user is issued a P12 client certificate. Nginx validates it on every request before forwarding to the backend. This is a temporary hardening measure while the application is pre-production; it will be removed after a security review. |

## AI

| | |
|---|---|
| **Provider** | Anthropic |
| **Service** | Claude Code, running in its own Docker container |
| **Role** | Powers AI-driven meal plan generation and recipe recommendations. Communicates with MariaDB directly, calling stored procedures for domain operations. |

## Infrastructure

| | |
|---|---|
| **Hosting** | Single VPS |
| **Containerisation** | Docker — each component (Nginx, MariaDB, Backend, Claude Code) runs as a separate Docker image on the same host. |
| **Deployment** | Ansible |
