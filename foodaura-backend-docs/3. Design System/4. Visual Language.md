# Visual Language

Foodaura looks like a well-lit kitchen at 4pm. Not a hospital. Not a tech app. Warm materials, natural light, real food.

The visual vibe was informed by a third-party food UI reference (warm cream/peach background, rounded cards, confident display type, soft pastel accents). We adopted the warmth, the rounded cards, and the soft pastel layering — but moved away from the loud red CTA toward a more grounded terracotta. Foodaura is calmer.

---

## Color Philosophy

### Background & surfaces
The page background is a warm off-white linen (`#FAF5EE`). Never pure white — white reads as clinical. Cards float on top at a slightly creamier value (`#FFFBF3`), creating depth without shadows.

Warm panels (section starters, active nav items, pressed states) use a soft peach (`#F6E5D6`). A muted sage tint (`#E4ECDF`) signals affirmation — a met target, an in-plan state.

### Primary — terracotta
`#C9603F`. Confident without being aggressive. Used for: the primary CTA button, the active marker on a selected day, focus rings. Appears in small doses. On dark surfaces, use `--primary-ink` (`#FFFBF3`) — never white.

### Accent — deep sage
`#5C7A5A`. Reserved for affirming context: a recipe in the plan, a preference match, a dietary goal met. Not used as a second brand colour — accent is always contextually meaningful.

### Neutrals
A stack of warm greys with a hint of brown, never cold blue-grey. The four foreground values (`--fg-1` through `--fg-4`) go from near-black to a light placeholder tone, all within the same warm family.

### Nutritional swatches
Four muted values that never compete with content:
- Clay (`#B97A56`) for protein
- Oat (`#C9A35C`) for carbs
- Olive (`#8A8C5C`) for fat
- Butter (`#C9925C`) for kcal

These appear as small dots next to labels. They are supporting context, not the feature.

### What we avoid
- Blue-purple gradients
- Medical white/green
- High-saturation accent colours
- Red used for anything except genuine errors (and even then, `--danger` is darker than the primary so they don't visually collide)

## Typography Philosophy

### Fraunces (display)
A humanist optical-size serif with subtle personality — ink traps, slight irregularity, warmth. Used for headings, recipe names, onboarding hero text, the big calorie number in recipe detail. The contrast between Fraunces and Jakarta Sans is the core typographic personality: one sophisticated, one practical.

### Plus Jakarta Sans (body)
Humanist, neutral, warm. Not the coldness of Inter, not the quirk of a personality font. At weight 400 it disappears into body text; at weight 600 it holds attention in buttons and labels without shouting.

### JetBrains Mono (quantities)
Used only where tabular alignment matters: the quantity column in the shopping list, ingredient amounts in the ingredient list. It signals "this is a measured value" without being a design statement.

### Type hierarchy in practice
A typical page reads: Fraunces 44px/52px for the h1 → Jakarta 16px/24px for body copy → Jakarta 13px/18px for captions → 11px uppercase tracked eyebrow for section labels. Never more than three sizes in a single viewport column.

## Spacing & Rhythm

Base unit: **4px**. All spacing values are multiples: 4, 8, 12, 16, 20, 24, 32, 40, 48, 64, 96.

The system uses generous padding. Cards have 24px interior padding. Page sections have 64–96px of vertical breathing room. The weekly plan grid uses 12px gutters between day columns; the shopping list uses 6–8px between rows.

**The rule:** if two things feel cramped, add one step up the scale. If they feel disconnected, drop one step.

---

## Layout

- Maximum content width: **1280px**, centered.
- The app uses a **240px persistent left rail** for primary nav on desktop. The `fa-app` grid is `240px 1fr`.
- Page content lives in `.fa-main` with 32px top padding, 40px horizontal, 64px bottom.
- A `.fa-page-head` flex row at the top of each interior page holds the h1 and CTA buttons.
- The weekly plan is always a **7-column grid** — it is the visual centrepiece of the app and should never wrap.

---

## Shadow System

All shadows use warm brown-tinted values (base `rgba(60, 38, 22, …)`) instead of neutral grey.

| Level | Value | Use |
|---|---|---|
| `--shadow-1` | `0 1px 2px rgba(60,38,22,.04)` | Cards at rest, rows |
| `--shadow-2` | `0 4px 12px rgba(60,38,22,.06)` | Hovered cards, sticky headers |
| `--shadow-3` | `0 12px 28px rgba(60,38,22,.08)` | Modals, popovers |

No inner shadows. No glows. No coloured shadows.

Cards have no visible border by default — the shadow provides depth. A 1px `--border` is added only when a card sits on the same surface colour.

---

## Motion

**Keep it quiet.** Animation communicates state, not personality.

- Color and opacity transitions: **180ms**, `--ease-soft` (`cubic-bezier(.2,.6,.2,1)`)
- Layout shifts: **220ms**, same easing
- Fast interactions (button press scale): **120ms**
- No bounces, no springs, no parallax, no wobble

**Hover on buttons:** `filter: brightness(0.96)` for primary — never a different hue.
**Hover on cards:** shadow deepens by one level + `translateY(-1px)`. Subtle lift.
**Press:** `filter: brightness(0.92)` + `scale(0.99)`. No ripple.
**Focus ring:** `0 0 0 3px rgba(201, 96, 63, 0.25)` — a soft terracotta halo, not a hard blue outline.

Skeleton loading states fade gently (0.6 → 1.0 alpha, 1.4s linear). No spinning loaders.

---

## Corner Radius

| Context | Radius |
|---|---|
| Cards, recipe tiles | 20px |
| Buttons, inputs, modal | 12–24px |
| Tags, pills, avatars | 999px |
| Recipe thumbnail images | 14px |

Radius reflects hierarchy: large containers are rounder, small inline elements are fully pillowed.

---

## Transparency & Blur

Blur is used in one place only: the sticky page header on mobile when content scrolls beneath it (`backdrop-filter: blur(12px)`). It is not decorative. It signals that the header is elevated above content.

Modals use a scrim at `rgba(42,31,20,.45)` with `backdrop-filter: blur(4px)` — enough to communicate depth, not enough to feel heavy.

---

## Photography Direction

The product ships with gradient placeholder thumbnails (`fa-thumb--peach`, `--sage`, `--oat`) until real recipe photography is available.

When photography arrives:
- **Warm, naturally lit, slightly overhead.** Real food on real surfaces: wooden boards, linen napkins, ceramic bowls.
- **No grain, no heavy filters, no black and white.**
- **No people.** Foodaura is about the food and the household, not lifestyle shoots.
- Images go full-bleed inside a rounded container. The `fa-recipe-hero` crops to a 16:7 aspect ratio.

---

## Icons

Lucide icons — outline, 1.6px stroke, rounded caps, 24×24 viewbox. `currentColor` so they inherit text colour. Default size in nav: 18px. In buttons: 16px. In empty-state heroes: 32px+.

The Lucide set is a substitution for a first-party icon set that doesn't exist yet. All icons used in the product are documented in the [[2. Design System]] reference.

---

## What We Avoid

- Pure white backgrounds
- Cold blue-grey neutrals
- High-saturation colours in any position
- Gradients on backgrounds or cards (only on image placeholder fills)
- Hard box-shadow colours (neutral grey)
- Decorative blur or parallax
- Bouncy or spring-based animations
- Stock photography, overhead plate-on-marble, heavy grain filters
