# Infrastructure & Deployment

## Hosting

The application runs on a single VPS. All components are co-located on the same host.

## Containers

Each component runs as an isolated Docker container on the same host:

| Container | Role |
|---|---|
| `nginx` | Reverse proxy — terminates HTTPS and validates P12 client certificates (mTLS) |
| `mariadb` | Primary datastore and domain logic host |
| `flyway` | One-shot migration runner — applies pending migrations and exits. Backend will not start until Flyway completes successfully. |
| `backend` | Go HTTP service (MVC, server-side rendering) |
| `ai` | AI service (Claude Code) — generates meal plans and recipe recommendations |

Containers communicate over a shared Docker network. Only Nginx exposes a port to the host network.

## Network Topology

```
Internet
    │
    ▼
Nginx (443, HTTPS + mTLS)
    │ plain HTTP
    ▼
Backend (Go)
    │              │
    │ MySQL         │ HTTP
    ▼              ▼
MariaDB         AI Service
                    │
                    │ MySQL
                    ▼
                MariaDB
```

Nginx is the sole public entry point. The backend, AI service, and MariaDB are not reachable from outside the Docker network.

## Internal Communication

| Source | Destination | Transport |
|---|---|---|
| Browser | Nginx | HTTPS + P12 client certificate (mTLS) |
| Nginx | Backend | Plain HTTP |
| Flyway | MariaDB | MySQL protocol (migrations only, one-shot) |
| Backend | MariaDB | MySQL protocol |
| Backend | AI service | HTTP |
| AI service | MariaDB | MySQL protocol (read-only) |

## TLS

Nginx terminates TLS. All traffic inside the Docker network travels over plain HTTP or the MySQL wire protocol — no internal TLS. Each user is issued a personal P12 client certificate for mTLS. See [Design Decisions § mTLS](5.%20Design%20Decisions.md) for why this is temporary.

## Deployment

Deployment is managed with Ansible. The playbook provisions the host, installs Docker, distributes configuration and secrets, and brings up the containers.

There is no CI/CD pipeline at this stage. Deployments are run manually by triggering the Ansible playbook from a local machine.
