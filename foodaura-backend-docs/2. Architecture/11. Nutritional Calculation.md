# Nutritional Calculation

Defines how `sp_profile_calculate_targets` derives a user's calorie and macro targets from their profile. This is the only place these rules are defined — the stored procedure implements them directly.

---

## Step 1 — Basal Metabolic Rate (BMR)

Formula: **Mifflin-St Jeor**

```
Male:   BMR = (10 × weight_kg) + (6.25 × height_cm) − (5 × age) + 5
Female: BMR = (10 × weight_kg) + (6.25 × height_cm) − (5 × age) − 161
```

BMR is the number of calories the body needs at complete rest. It is the baseline before activity and goal adjustments.

---

## Step 2 — Total Daily Energy Expenditure (TDEE)

```
TDEE = BMR × activity_multiplier
```

| `activity_level` | Multiplier | Description |
|---|---|---|
| `sedentary` | 1.2 | Little or no exercise |
| `light` | 1.375 | Light exercise 1–3 days/week |
| `moderate` | 1.55 | Moderate exercise 3–5 days/week |
| `active` | 1.725 | Hard exercise 6–7 days/week |
| `very_active` | 1.9 | Very hard exercise or physical job |

TDEE is the number of calories needed to maintain current weight at the given activity level.

---

## Step 3 — Calorie Target

```
target_calories = TDEE + goal_adjustment
```

| `goal` | Adjustment | Intent |
|---|---|---|
| `lose_weight` | −500 kcal | ~0.5 kg/week deficit |
| `maintain` | 0 | No change |
| `build_muscle` | +300 kcal | Lean bulk |
| `eat_better` | 0 | Quality focus, no calorie shift |

---

## Step 4 — Macro Targets

Macros are derived from `target_calories` and `weight_kg`. Order matters: protein is set first, fat second, carbs absorb the remainder.

### Protein

```
protein_g = protein_per_kg × weight_kg
```

| `goal` | `protein_per_kg` |
|---|---|
| `lose_weight` | 1.6 g/kg — preserves muscle during deficit |
| `maintain` | 1.2 g/kg |
| `build_muscle` | 2.0 g/kg |
| `eat_better` | 1.2 g/kg |

### Fat

```
fat_g = ROUND((target_calories × fat_ratio) / 9)
```

| `goal` | `fat_ratio` |
|---|---|
| `lose_weight` | 0.25 |
| `maintain` | 0.30 |
| `build_muscle` | 0.25 |
| `eat_better` | 0.30 |

### Carbohydrates

```
carbs_g = ROUND((target_calories − (protein_g × 4) − (fat_g × 9)) / 4)
```

Carbs take whatever calories remain after protein and fat are allocated.

---

## Worked Example

**Profile:** Female, 28 years, 65 kg, 165 cm, light activity, lose_weight

| Step | Calculation | Result |
|---|---|---|
| BMR | (10×65) + (6.25×165) − (5×28) − 161 | 1380 kcal |
| TDEE | 1380 × 1.375 | 1898 kcal |
| Target calories | 1898 − 500 | 1398 kcal |
| Protein | 1.6 × 65 | 104 g |
| Fat | (1398 × 0.25) / 9 | 39 g |
| Carbs | (1398 − 416 − 351) / 4 | 158 g |

---

## Notes

- All values are **rounded to the nearest integer** before storage.
- Targets are stored in `nutritional_targets` and recalculated automatically by `trg_nutritional_profiles_after_upsert` whenever the profile changes.
- Targets are used internally for recipe scaling and meal plan generation. They are not surfaced as a daily tracking interface — see [Features § Nutritional Context](../1.%20Foodaura/4.%20Features.md).
